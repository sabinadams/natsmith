# Connection flags

These flags belong to the **`migrate`** command group. Pass them when running `natsmith migrate kv` or `natsmith migrate objects` — not for top-level help (`natsmith -h`) or other future commands.

| Flag | Required | Example |
|------|----------|---------|
| `-source-url` | yes | `nats://source.example.com:4222` |
| `-dest-url` | yes | `nats://dest.example.com:4222` |
| `-source-creds` | when source uses `.creds` | `/path/to/source.creds` |
| `-dest-creds` | when destination uses `.creds` | `/path/to/dest.creds` |

They are persistent flags on `migrate`, so you can place them before or after the subcommand:

```bash
natsmith migrate kv -source-url … -dest-url … -dry-run
natsmith migrate kv -source-url … -dest-url … -dry-run   # same flags
```

## Shared migrate flags

Both `migrate kv` and `migrate objects` accept:

| Flag | Description |
|------|-------------|
| `-bucket` | Comma-separated buckets to migrate (default: all) |
| `-omit` | Comma-separated buckets to skip |
| `-dry-run` | List buckets and records without writing |
| `-skip-existing` | Skip records that already exist on destination |
| `-workers` | Concurrent workers (1–64, default 1) |
| `-timeout` | Per-request NATS timeout (default 30s) |
| `-no-progress` | Plain log output (useful in CI) |

Use `-bucket` to select buckets, `-omit` to exclude them, or both (`-bucket` first, then `-omit` removes from that set).
