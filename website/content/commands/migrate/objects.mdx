# migrate objects

```bash
natsmith migrate objects [flags]
```

Copy object blobs and recreate link objects in a second pass.

| Behavior | Detail |
|----------|--------|
| Destination buckets | Opens existing buckets only — does not create or set descriptions |
| Scan | Full `OBJ_<bucket>` stream for meta, then `GetInfo` probe per candidate |
| Omitted objects | Meta exists but data is not retrievable (common on legacy buckets) |
| Timeouts | Each copy uses at least 5 minutes; raise `--timeout` for large files |
| Concurrency | Lower `--workers` (e.g. 8) for buckets with very large objects |

## Flags

Inherits all [migrate](/commands/migrate) flags. No object-specific flags.

## Output

**Dry-run:** `listed`, `meta-active`, `meta-omitted` per bucket.

**Full run:** `listed`, `migratable`, `omitted`, then `N/N copied` with optional `(N failed)`.

No verify mode (unlike KV).

## Examples

### Dry-run all buckets

```bash
natsmith migrate objects --dry-run \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --workers 16
```

### Migrate all buckets

```bash
natsmith migrate objects \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --workers 16 \
  --timeout 5m
```

### Migrate one bucket

```bash
natsmith migrate objects \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --bucket my-bucket \
  --workers 16 \
  --timeout 5m
```
