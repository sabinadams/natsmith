# migrate

```bash
natsmith migrate <subcommand> [flags]
```

Copy JetStream KV buckets and object stores from a source cluster to a destination.

## Subcommands

| Command | Description |
|---------|-------------|
| [`kv`](/commands/migrate/kv) | Copy KV buckets (`ListKeys` → `Get` → `Put`) |
| [`objects`](/commands/migrate/objects) | Copy object store buckets |

## Global flags

Persistent on `migrate` (apply to all subcommands):

| Flag | Default | Description |
|------|---------|-------------|
| `--source-context` | — | Source [NATS CLI context](https://docs.nats.io/using-nats/nats-tools/nats_cli/context) (required) |
| `--dest-context` | — | Destination context (required) |
| `--bucket` | all | Comma-separated bucket filter |
| `--omit` | — | Comma-separated buckets to skip |
| `--dry-run` | `false` | List/count only; no reads or writes |
| `--skip-existing` | `false` | Skip keys/objects already on destination |
| `--workers` | `1` | Parallel workers, 1–64 |
| `--timeout` | `30s` | Per-request NATS timeout |
| `--no-progress` | `false` | Plain stderr (no progress UI) |

## Behavior

- **Source:** read-only; never deletes data.
- **Destination:** writes matching records; does not remove extra keys/objects.
- **Buckets:** must already exist on destination (e.g. Terraform).

For KV strategy (migrate vs backup/restore), see [KV migration](/guides/kv-migration).

## Example

```bash
natsmith migrate kv --dry-run \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated
```
