# migrate kv

```bash
natsmith migrate kv [flags]
```

Copy or verify live KV keys from source to destination using the KV API.

**Pipeline:** `ListKeys` → parallel `Get` → parallel `Put` → optional inline verify.

## Flags

Inherits all [migrate](/commands/migrate) flags, plus:

| Flag | Default | Description |
|------|---------|-------------|
| `--verify` | `true` | Byte-for-byte check after copy |
| `--verify-only` | `false` | Compare without writing |
| `--failures-file` | — | Append verify issues to a log file |

## Output

| Term | Meaning |
|------|---------|
| migratable | Live keys listed on source |
| ghost skipped | Keys in `ListKeys` but missing on `Get` (skipped, run continues) |

Success: `N/N migratable copied` and `verify: N ok, 0 missing, 0 mismatch, 0 dest-only`.

## Examples

### Dry-run all buckets

```bash
natsmith migrate kv --dry-run \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated
```

### Migrate all buckets

```bash
natsmith migrate kv \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --workers 32 \
  --failures-file kv-failures.log
```

### Migrate one bucket

```bash
natsmith migrate kv \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --bucket schema \
  --workers 32 \
  --failures-file kv-failures.log
```

### Verify only

```bash
natsmith migrate kv --verify-only \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --bucket schema \
  --workers 16
```

### Resume an interrupted run

```bash
natsmith migrate kv \
  --source-context sandbox-legacy \
  --dest-context sandbox-dedicated \
  --skip-existing \
  --workers 32
```

## Notes

- Keys processed in batches — memory stays bounded for very large buckets.
- Destination-only scan skipped when bucket has &gt; 1M keys (inline verify still runs).
- For millions of keys, prefer [backup/restore](/guides/kv-migration).
